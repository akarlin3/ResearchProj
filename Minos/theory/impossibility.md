# Theorem 2, part (i) — Label-free undetectability of the hidden channel

> # ⚠️ REQUIRES HUMAN PROOF-REVIEW — NOT MACHINE-VERIFIED ⚠️
>
> This is a **drafted** data-processing / sufficiency argument, not algebra a symbolic engine
> checked. The numeric companion (`detectability.py`) only verifies the *structural premise* —
> that the hidden shift leaves the observable law `P_O` (and hence the monitor `M`) bit-identical
> (`M(δ_hid=0) = M(δ_hid=0.20) = 0.007248` exactly). The **conclusion** below (best detection
> AUC = ½) is the human-reviewable claim. It is consistent with the v3 simulation
> (`RESULTS_C.md`: hidden-channel detection AUC = `0.500`), but consistency is not proof.

---

## 1. Setup (in the locked model of `THEORY_MODEL.md`)

A deployment batch is generated by `realise_deploy` (`generative.py:172`) from base draws with reported
centres `c_i` and standardised skew errors `u_i`:

```
observable channel (δ_obs):   μ_i = c_i − β·s·δ_obs        (reported point)
hidden channel     (δ_hid):   θ_i = c_i + β·s·δ_hid + s·u_i   (truth)
```

A **label-free monitor** is any measurable map `M = f(O)` of the *observable* batch
`O = {μ_i}_{i=1..n}` (it may also use the known, fixed `cfg`/utility and the frozen calibration
reference, but it may **not** read the truths `{θ_i}` — that is the no-label discipline asserted in
`monitor.py:16`). A **detector** is any decision rule that, from `M` (equivalently from `O`), declares
the batch *stale* or *fresh*.

Consider the pure hidden channel: `δ_obs = 0`, and compare two regimes,

* **fresh:** `δ_hid = 0` (the calibration distribution; stale-correction regret `R ≈ 0`),
* **stale:** `δ_hid = δ > 0` (the same regret-inducing discrepancy as a matched observable shift, by
  construction — `RESULTS_C.md` shows the two channels induce comparable `R`; `R > tol`).

We claim no label-free detector separates fresh from stale better than chance.

## 2. Claim

> **Theorem 2(i).** For the hidden channel (`δ_obs = 0`), every label-free statistic `M = f(O)` has
> identical law under *fresh* and *stale*. Consequently every detector built from `O` has
> true-positive rate equal to its false-positive rate at every operating point; its ROC is the
> diagonal and its detection AUC for the event `{R > tol}` is exactly **½**.

## 3. Proof (data-processing / sufficiency)

**Step 1 — the observable law is invariant to the hidden channel.**
With `δ_obs = 0`, the observable batch is `O = {μ_i} = {c_i}`, which depends only on the reported
centres `c_i`. The hidden parameter `δ_hid` enters **only** `θ_i = c_i + β·s·δ_hid + s·u_i`, and `θ` is
not part of `O`. Hence the joint distribution of `O` is the same under both regimes:

```
P_O^{stale} = P_O^{fresh} =: P_O.
```

(The simulation realises this exactly: `realise_deploy` computes `μ` without reference to `δ_hid`, so
the reported points are bit-identical across `δ_hid` — verified in `detectability.py`.)

**Step 2 — any function of `O` inherits the invariance.**
`M = f(O)` is a measurable function of `O` alone. The pushforward of a fixed measure through a fixed
map is fixed, so

```
Law(M ∣ stale) = f_# P_O^{stale} = f_# P_O^{fresh} = Law(M ∣ fresh).
```

Equivalently, the data-processing inequality gives `I(M ; δ_hid) ≤ I(O ; δ_hid) = 0` (the mutual
information vanishes because `P_O` does not depend on `δ_hid`), so `M` carries **zero** information
about which regime produced the batch.

**Step 3 — no detector has power.**
By Neyman–Pearson, the most powerful test of *fresh* vs *stale* from `O` thresholds the likelihood
ratio `dP_O^{stale}/dP_O^{fresh}`. By Step 1 this ratio is `≡ 1`. Therefore for any threshold `m*`,

```
TPR(m*) = P(M > m* ∣ stale) = P(M > m* ∣ fresh) = FPR(m*),
```

so the ROC curve is the diagonal `TPR = FPR` and the area under it is `AUC = ½`. Because Step 2 holds
for *every* `f`, no choice of label-free statistic — the default utility-divergence `M`, the
action-divergence variant, or any other function of `{μ}` — can do better than chance. ∎

## 4. Scope, assumptions, and the honest caveat

* **What is assumed.** The hidden channel leaves the observable law `P_O` **exactly** invariant. In the
  Minos-Core split this is true *by construction* (`δ_hid` perturbs `θ` only). The theorem is sharp for
  this idealisation.
* **What is not claimed.** Real deployment shifts need not factor so cleanly: a physically "hidden"
  change may leak into observables (e.g. correlated acquisition features), in which case `P_O` *does*
  move and the observable-channel bound of part (ii) applies to whatever leaks. The theorem bounds the
  *genuinely* hidden component — the part that, by definition, leaves every observable summary
  untouched.
* **Why it matters.** Parts (i)+(ii) together say: a label-free monitor can be made arbitrarily good on
  the observable component (`R_obs ≤ L·δ`, part ii) and is provably powerless on the hidden component
  (AUC = ½, part i). The hidden component induces *the same regret* yet is invisible. This is the
  principled argument for **periodic labeled repeatability spot-checks** alongside the monitor: only a
  statistic that reads `θ` (a label) can see the hidden channel, precisely because every function of
  the observables cannot.

## 5. For the reviewer

Points to scrutinise:
1. Is "label-free monitor = measurable function of `O = {μ_i}` (plus fixed known constants)" the right
   formalisation of the no-label discipline? (It matches the code's assertion, but the prose definition
   should be pinned down for the paper.)
2. Step 1's exact invariance is a modelling choice. The theorem's force depends on it; the paper should
   state it as the definition of "hidden" rather than a derived property.
3. The AUC = ½ statement is for the *population* ROC. The v3 empirical AUC = `0.500` is a finite-sample
   estimate; the proof concerns the idealised law. No finite-sample concentration claim is made here.
